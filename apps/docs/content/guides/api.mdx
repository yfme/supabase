Supabase 直接从您的数据库架构自动生成 API，允许您通过浏览器中的 RESTful 接口连接到您的数据库。

API 是从您的数据库自动生成的，旨在让您尽快构建应用，而无需编写一行代码。

您可以直接从浏览器使用它们（两层架构），或者作为您自己的 API 服务器的补充（三层架构）。

## 功能 [#rest-api-overview]

Supabase 使用 [PostgREST](https://postgrest.org/) 提供 RESTful API。这是一个建立在 Postgres 之上的非常轻量级的 API 层。
它通过 URL `https://<project_ref>.supabase.co/rest/v1/` 暴露了您需要的所有 CRUD API 功能。

REST 接口是从您的数据库架构自动生成的，它具有以下特点：

- **即时且自动生成。** <br />当您更新数据库时，变更会立即通过 API 访问。
- **自我文档化。** <br />Supabase 在仪表板中生成文档，随着您进行数据库更改而更新。
- **安全。** <br />API 配置为与 PostgreSQL 的行级安全性一起工作，置于启用了密钥认证的 API 网关后面。
- **快速。** <br />我们的基本读取基准测试比 Firebase 快 300% 以上。API 是 Postgres 之上的一个非常薄的层，Postgres 完成了大部分繁重的工作。
- **可扩展。** <br />API 可以服务于数千个同时请求，并且适用于无服务器工作负载。

反映的 API 旨在保留尽可能多的 Postgres 功能，包括：

- 基本的 CRUD 操作（创建/读取/更新/删除）
- 表/视图之间任意深度的关系，返回表类型的函数也可以嵌套相关表/视图。
- 适用于 Postgres 视图、物化视图和外部表
- 适用于 Postgres 函数
- 用户定义的计算列和计算关系
- Postgres 安全模型 - 包括行级安全性、角色和授权。

REST API 将所有请求解析为单个 SQL 语句，从而导致响应时间快和高吞吐量。

参考资料：

- [文档](https://postgrest.org/)
- [源代码](https://github.com/PostgREST/postgrest)
