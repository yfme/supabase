---
title: Supabase 队列
subtitle: '在 Postgres 中具有保证交付的持久消息队列'
---

Supabase 队列是一个基于 [pgmq 数据库扩展](https://github.com/tembo-io/pgmq) 构建的具有保证交付功能的 Postgres 原生持久消息队列系统。它为开发人员提供了一种无缝的方式来在后台持久化和处理消息，同时提高应用程序和服务的弹性和可扩展性。

队列将 Postgres 的可靠性与 Supabase 平台和开发者体验的简洁性结合起来，使开发人员能够零配置地管理后台任务。

## 功能特性

- **Postgres 原生**
  <br />
  构建在 `pgmq` 数据库扩展之上，使用任何 Postgres 工具创建和管理队列。
- **保证消息交付**
  <br />
  添加到队列的消息保证会被交付给您的消费者。
- **消息恰好一次交付**
  <br />在可自定义的可见性窗口内，一条消息恰好传递一次给消费者。
- **消息持久性和归档**
  <br />
  消息存储在 Postgres 中，您可以选择将其归档以用于分析或审计目的。
- **精细授权**
  <br />
  使用 API 权限和行级安全性 (RLS) 策略控制客户端消费者对队列的访问。
- **队列管理和监控**
  <br />
  在 Supabase 仪表板中创建、管理和监控队列和消息。
