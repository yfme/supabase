随着向量表规模的增长，您可能需要添加索引来加速查询。在没有索引的情况下，系统将执行顺序扫描，这在记录数量庞大时会消耗大量资源。

## 选择索引

目前 `pgvector` 支持两种类型的索引：

- [HNSW](/docs/guides/ai/vector-indexes/hnsw-indexes)
- [IVFFlat](/docs/guides/ai/vector-indexes/ivf-indexes)

通常我们推荐使用 [HNSW](/docs/guides/ai/vector-indexes/hnsw-indexes)，因为它具有更好的[性能](https://supabase.com/blog/increase-performance-pgvector-hnsw#hnsw-performance-1536-dimensions)和[对数据变化的鲁棒性](/docs/guides/ai/vector-indexes/hnsw-indexes#when-should-you-create-hnsw-indexes)。

## 距离运算符

索引可用于提高使用各种距离度量的最近邻搜索性能。`pgvector` 包含 3 个距离运算符：

| 运算符 | 描述 | [**运算符类**](https://www.postgresql.org/docs/current/sql-createopclass.html) |
| -------- | ---------------------- | ------------------------------------------------------------------------------------ |
| `<->` | 欧几里得距离 | `vector_l2_ops` |
| `<#>` | 负内积 | `vector_ip_ops` |
| `<=>` | 余弦距离 | `vector_cosine_ops` |

目前可以索引最多 2,000 维的向量。
