Supabase Auth 使在您的应用中实现身份验证和授权变得简单。我们提供客户端 SDK 和 API 端点来帮助您创建和管理用户。

您的用户可以使用多种流行的身份验证方法，包括密码、魔法链接、一次性密码 (OTP)、社交登录和单点登录 (SSO)。

## 关于身份验证和授权

身份验证和授权是任何身份验证系统的核心职责。

- **身份验证**意味着确认用户是他们所声称的身份。
- **授权**意味着检查用户被允许访问哪些资源。

Supabase Auth 使用 [JSON Web Tokens (JWTs)](/docs/guides/auth/jwts) 进行身份验证。Auth 与 Supabase 的数据库功能集成，使得使用[行级安全性 (RLS)](/docs/guides/database/postgres/row-level-security) 进行授权变得简单。

## Supabase 生态系统

您可以将 Supabase Auth 作为独立产品使用，但它也被构建为与 Supabase 生态系统集成。

Auth 在后台使用您项目的 Postgres 数据库，将用户数据和其他 Auth 信息存储在一个特殊的模式中。您可以使用触发器和外键引用将这些数据连接到您自己的表中。

Auth 还允许访问您数据库自动生成的 [REST API](/docs/guides/api)。当使用 Supabase SDK 时，您的数据请求会自动附带用户的 Auth Token。当与 [RLS 策略](/docs/guides/database/postgres/row-level-security) 一起使用时，Auth Token 可以在行级别上限定数据库访问范围。
