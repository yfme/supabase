Supabase 文档中使用的术语和缩写的定义。

## 访问令牌（Access token）

访问令牌是一种短期（通常不超过 1 小时）的令牌，用于授权客户端访问服务器上的资源。它以 [JSON Web 令牌 (JWT)](#json-web-token-jwt) 的形式出现。

## 身份验证（Authentication）

身份验证（通常缩写为 `authn.`）是验证用户身份的过程。验证用户身份可以通过多种方式进行：

1. 询问用户所知道的信息。例如：密码、密码短语。
2. 检查用户是否能够访问他们拥有的东西。例如：电子邮件地址、电话号码、硬件密钥、恢复码。
3. 确认用户具有某些生物特征。例如：指纹、特定的面部结构、虹膜印记。

## 认证器应用（Authenticator app）

认证器应用生成基于时间的一次性密码（TOTPs）。这些密码基于一个长且难以猜测的秘密字符串生成。最初通过扫描二维码将秘密传递给应用程序。

## 授权（Authorization）

授权（通常缩写为 `authz.`）是验证某个身份是否被允许访问资源的过程。授权通常通过验证访问令牌来完成。

## 身份提供商（Identity provider）

身份提供商是允许第三方应用程序在不交换密码的情况下识别用户的软件或服务。没有身份提供商，社交登录和企业单点登录将无法实现。

社交登录平台通常使用 OAuth 协议，而企业单点登录基于 OIDC 或 SAML 协议。

## JSON Web 令牌（JWT）

[JSON Web 令牌](https://jwt.io/introduction)是一种数据结构，以字符串形式表示，通常包含有关用户的身份和授权信息。它编码了有关其生命周期的信息，并使用加密密钥进行签名，使其具有防篡改性。

访问令牌是 JWT，通过检查它们包含的信息，您可以允许或拒绝对资源的访问。行级安全策略基于 JWT 中存在的信息。

## JWT 签名密钥（JWT signing secret）

Supabase 发行的 JWT 使用 HMAC-SHA256 算法进行签名。签名中使用的密钥称为 JWT 签名密钥。您不应该与不信任的人或事物共享此密钥，也不应该公开发布它。任何能够访问该密钥的人都可以创建任意的 JWT。

## 多因素认证（MFA 或 2FA）

多因素认证是通过组合多种因素来验证用户身份的过程：用户知道的东西、用户拥有的东西或用户本身的特征。

## 随机数（Nonce）

Nonce 表示仅使用一次的数字。但实际上，它是一个唯一且难以猜测的字符串，用于安全地初始化协议或算法，或在各种重放攻击形式中检测滥用行为。

## OAuth

OAuth 是一种协议，允许第三方应用程序请求和接收来自其用户的授权。它通常用于实现社交登录，并作为 OIDC 协议中企业单点登录的基础。应用程序可以请求不同级别的访问权限，包括基本的用户识别信息，如姓名、电子邮件地址和用户 ID。

## OIDC

OIDC 代表 OpenID Connect，是一种使企业能够实现单点登录的协议。OIDC 基于现代 Web 技术，如 OAuth 和 JSON Web 令牌。它通常用于替代较旧的 SAML 协议。

## 一次性密码（OTP）

一次性密码是发送到设备（如电话号码）或由设备或应用程序生成的短、随机生成且难以猜测的密码或代码。

## 密码哈希函数（Password hashing function）

密码哈希函数是专门设计的算法，允许 Web 服务器验证密码而无需按原样存储它。与其他由安全随机数生成器生成的难以猜测的字符串不同，密码是由用户选择的，通常容易被攻击者猜到。这些算法能够减慢并使攻击者猜测密码的成本大大增加。

一般接受的三种密码哈希函数是：Argon2、bcrypt 和 scrypt。

## 密码强度（Password strength）

密码强度是衡量密码难以猜测程度的指标。简单的测量包括计算给定密码中使用的字符类型的可能性数量。例如，仅由字母组成的密码比由字母和数字组成的密码变化更少。更好的测量方法包括寻找与单词、短语或已知密码的相似性等策略。

## PKCE

Proof Key for Code Exchange（验证密钥代码交换）是 OAuth 协议的扩展，可在应用程序（Web 应用程序、单页应用程序或移动应用程序）和授权服务器之间安全地交换刷新令牌和访问令牌。它用于可能被第三方（如操作系统中运行的其他应用程序）截获刷新令牌和访问令牌交换的地方。这在移动设备上是一个常见问题，操作系统可能会将 URL 交给其他应用程序。这有时也可能在单页应用程序中被利用。

## 提供商刷新令牌（Provider refresh token）

提供商刷新令牌是由第三方身份提供商发行的刷新令牌，可用于刷新返回的提供商令牌。

## 提供商令牌（Provider tokens）

提供商令牌是由第三方身份提供商发行的长期有效的令牌。这些令牌由社交登录服务（例如 Google、Twitter、Apple、Microsoft）发行，并在这些平台上唯一标识用户。

## 刷新令牌（Refresh token）

刷新令牌是一种长期有效（在大多数情况下具有无限生命周期）的令牌，旨在被存储并只交换一次以获取新的刷新令牌和访问令牌。一旦刷新令牌被交换，它就会失效，并且不能再次交换。但在实践中，刷新令牌可以在短时间窗口内多次交换。

## 刷新令牌流程（Refresh token flow）

刷新令牌流程是基于有效的刷新令牌发行新的刷新令牌和访问令牌的机制。它用于延长应用程序的授权访问。持续使用的应用程序将在访问令牌过期之前调用刷新令牌流程。

## 重放攻击（Replay attack）

重放攻击是攻击者窃取或拦截敏感信息，然后尝试再次使用它（即重放）以企图破坏系统的行为。通常可以通过适当使用随机数来缓解重放攻击。

## 行级安全策略（RLS）

行级安全策略是 Postgres 数据库中的特殊对象，限制了返回给客户端的可用操作或数据。RLS 策略使用 JWT 中包含的信息来识别用户以及他们被允许执行或查看的操作和数据。

## SAML

SAML 代表安全断言标记语言（Security Assertion Markup Language），是一种使企业能够实现单点登录的协议。SAML 发明于 2000 年代初，基于 XML 技术。它是企业单点登录的事实标准，尽管更新的 OIDC（OpenID Connect）协议正在获得普及。

## 会话（Session）

会话或身份验证会话是将已验证的用户身份绑定到 Web 浏览器的概念。会话通常是长期存在的，可以通过用户注销来终止。访问令牌和刷新令牌对在浏览器中代表一个会话，它们存储在本地存储或作为 cookie。

## 单点登录（SSO）

单点登录允许企业集中管理账户和应用程序访问。它们使用身份提供商软件或服务将员工信息组织在目录中，并通过 OIDC 或 SAML 协议将这些账户与应用程序连接起来。

## 基于时间的一次性密码（TOTP）

基于时间的一次性密码是从密钥（通常是移动设备中的应用程序，如 Google Authenticator、1Password）以固定时间间隔生成的一次性密码。
