---
title: 部署
---

部署您的应用使其上线并可供用户访问。通常，您至少会将应用部署到两个环境：用于用户的生产环境和（一个或多个）用于开发人员的暂存或预览环境。

Supabase 提供了几种环境管理和部署选项。

## 环境管理

您可以为 Supabase 维护单独的开发、暂存和生产环境：

- **开发**：使用 [Supabase CLI](/docs/guides/local-development) 通过本地 Supabase 技术栈进行开发。
- **暂存**：使用[分支](/docs/guides/deployment/branching)创建暂存或预览环境。您可以使用持久分支进行长期暂存设置，或使用临时分支进行短期预览（通常与拉取请求相关联）。
- **生产**：如果您启用了分支功能，您可以使用 Supabase GitHub 集成在合并拉取请求时自动推送迁移文件。或者，您可以使用 Supabase CLI 设置自己的持续部署流程。

<Admonition type="tip" title="自托管">

请参阅[自托管指南](/docs/guides/self-hosting)了解如何托管您自己的 Supabase 技术栈。

</Admonition>

## 部署

您可以使用以下方式自动化部署：

- [Supabase GitHub 集成](/dashboard/project/_/settings/integrations)（启用分支功能）
- 在您自己的持续部署流程中使用 [Supabase CLI](/docs/guides/local-development)
- [Supabase Terraform 提供商](/docs/guides/deployment/terraform)
