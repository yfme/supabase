这是 Supabase 为每个项目提供的功能的非详尽列表。

## 数据库

### Postgres 数据库

每个项目都是一个完整的 Postgres 数据库。[文档](/docs/guides/database)。

### 向量数据库

将向量嵌入存储在您的其他数据旁边。[文档](/docs/guides/ai)。

### 通过 PostgREST 自动生成的 REST API

无需编写一行代码，即可从数据库自动生成 RESTful API。[文档](/docs/guides/api#rest-api-overview)。

### 通过 pg_graphql 自动生成的 GraphQL API

使用我们自定义的 Postgres GraphQL 扩展创建快速的 GraphQL API。[文档](/docs/guides/graphql/api)。

### 数据库 Webhook

使用 Webhook 将数据库更改发送到任何外部服务。[文档](/docs/guides/database/webhooks)。

### 机密和加密

使用我们的 Postgres 扩展 Supabase Vault 加密敏感数据和存储机密。[文档](/docs/guides/database/vault)。

## 平台

### 数据库备份

项目每天进行备份，可选升级到时间点恢复。[文档](/docs/guides/platform/backups)。

### 自定义域名

为 Supabase API 设置白标，为用户创建品牌体验。[文档](/docs/guides/platform/custom-domains)。

### 网络限制

限制可连接到数据库的 IP 范围。[文档](/docs/guides/platform/network-restrictions)。

### SSL 强制

强制 Postgres 客户端通过 SSL 连接。[文档](/docs/guides/platform/ssl-enforcement)。

### 分支

使用 Supabase 分支测试和预览更改。[文档](/docs/guides/platform/branching)。

### Terraform 提供商

通过 Terraform（一种基础设施即代码工具）管理 Supabase 基础设施。[文档](/docs/guides/platform/terraform)。

### 只读副本

跨多个区域部署只读数据库，降低延迟并提高资源管理。[文档](/docs/guides/platform/read-replicas)。

### 日志排放

将 Supabase 日志导出到第三方提供商和外部工具。[文档](/docs/guides/platform/log-drains)。

## Studio

### Studio 单点登录

通过 SSO 登录 Supabase 仪表板。[文档](/docs/guides/platform/sso)。

<br />

## 实时功能

### Postgres 变更

通过 WebSocket 接收数据库变更。[文档](/docs/guides/realtime/postgres-changes)。

### 广播

通过 WebSocket 在已连接用户之间发送消息。[文档](/docs/guides/realtime/broadcast)。

### 在线状态

同步用户间的共享状态，包括在线状态和输入指示器。[文档](/docs/guides/realtime/presence)。

## 认证

### 电子邮件登录

为应用程序或网站构建电子邮件登录。[文档](/docs/guides/auth/auth-email)。

### 社交登录

提供社交登录——从 Apple 到 GitHub 再到 Slack 的所有服务。[文档](/docs/guides/auth/social-login)。

### 电话登录

使用第三方 SMS 提供商提供电话登录。[文档](/docs/guides/auth/phone-login)。

### 无密码登录

通过魔术链接为应用程序或网站构建无密码登录。[文档](/docs/guides/auth/auth-magic-link)。

### 通过行级安全性授权

使用 Postgres 策略控制每个用户可以访问的数据。[文档](/docs/guides/database/postgres/row-level-security)。

### CAPTCHA 保护

为登录、注册和密码重置表单添加 CAPTCHA。[文档](/docs/guides/auth/auth-captcha)。

### 服务器端认证

用于在流行的服务器端语言和框架（如 Next.js、SvelteKit 和 Remix）中实现用户认证的辅助工具。[文档](/docs/guides/auth/server-side)。

## 功能状态

Supabase 功能有 4 种不同状态 - 私有 Alpha、公共 Alpha、Beta 和正式可用。

### 私有 alpha

功能最初作为私有 alpha 版本推出，以收集社区反馈。要加入我们的早期访问计划，请发送电子邮件至 product-ops@supabase.io。

### 公共 alpha

Alpha 阶段表示 API 可能在未来发生变化，而不是服务不稳定。尽管 [uptime 服务级别协议](https://supabase.com/sla) 不涵盖 Alpha 中的产品，但我们会尽最大努力使服务尽可能稳定。

### Beta

Beta 版功能已由外部渗透测试人员进行安全问题测试。API 保证稳定，并且对于破坏性变更有严格的沟通流程。

### 正式可用

除了 Beta 要求外，正式可用的功能还受 [uptime SLA](https://supabase.com/sla) 的保障。
