## 本地开发

使用 Supabase 进行本地开发允许您在本地机器上的自包含环境中处理项目。本地工作有几个优势：

1. 更快的开发：您可以立即更改并查看结果，无需等待远程部署。
2. 离线工作：即使没有互联网连接，您也可以继续开发。
3. 成本效益：本地开发是免费的，不会消耗您项目的配额。
4. 增强隐私：开发期间敏感数据保留在您的本地机器上。
5. 轻松测试：您可以在不影响生产环境的情况下尝试不同的配置和功能。

要开始本地开发，您需要安装 [Supabase CLI](#cli) 和 Docker。Supabase CLI 允许您启动和管理本地 Supabase 技术栈，而 Docker 用于运行必要的服务。

设置完成后，您可以初始化一个新的 Supabase 项目，启动本地技术栈，并开始使用本地 Supabase 服务开发您的应用程序。这包括访问本地 Postgres 数据库、身份验证、存储和其他 Supabase 功能。

## CLI

Supabase CLI 是一个强大的工具，使开发人员能够直接从终端管理他们的 Supabase 项目。它提供了一套用于各种任务的命令，包括：

- 设置和管理本地开发环境
- 为您的数据库架构生成 TypeScript 类型
- 处理数据库迁移
- 管理环境变量和秘密
- 将您的项目部署到 Supabase 平台

通过 CLI，您可以简化开发工作流程，自动化重复任务，并保持不同环境之间的一致性。它是本地开发和 CI/CD 流程的基本工具。

有关更多信息，请参阅 [CLI 入门指南](/docs/guides/local-development/cli/getting-started)。
